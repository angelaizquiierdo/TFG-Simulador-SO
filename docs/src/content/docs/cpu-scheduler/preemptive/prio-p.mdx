## Descripcion

El algoritmo de **Prioridad expropiativo** selecciona en cada tick el proceso con menor
numero de prioridad entre los que estan en la cola lista. Un numero menor indica mayor
prioridad (p. ej., prioridad 1 es mas urgente que prioridad 3).

A diferencia de la version no expropiativa, si llega un proceso con mayor prioridad que
el proceso actualmente en CPU, este es expropiado de inmediato.

- Garantiza que los procesos urgentes accedan a la CPU rapidamente.
- Puede causar inanicion de procesos de baja prioridad.
