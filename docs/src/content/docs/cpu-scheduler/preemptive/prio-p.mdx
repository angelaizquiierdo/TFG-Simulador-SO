# Prioridad expropiativa

En cada tick selecciona el proceso con **mayor prioridad** (número más bajo).
Si llega un proceso con mayor prioridad que el actual en CPU, lo expropia.

## Características

- **Modo**: expropiativo (`on-better`)
- **Criterio de selección**: menor `priority` (número)
- **Campo requerido**: `priority` en cada proceso
- **Starvation**: posible en procesos de baja prioridad

## Escenario de ejemplo

| Proceso | Llegada | Ráfaga | Prioridad |
|---------|---------|--------|-----------|
| P1      | 0       | 5      | 3         |
| P2      | 1       | 3      | 2         |
| P3      | 2       | 2      | 1         |

Diagrama de Gantt esperado: P1[0–1], P2[1–2], P3[2–4], P2[4–6], P1[6–10]
