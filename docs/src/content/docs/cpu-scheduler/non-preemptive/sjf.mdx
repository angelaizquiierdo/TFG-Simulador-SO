## Descripción

**SJF** (Shortest Job First) selecciona siempre el proceso con la **ráfaga más corta** de entre los procesos listos. Es **no expropiativo**: una vez que un proceso ocupa la CPU, la mantiene hasta que termina.

- **Ventaja:** minimiza el tiempo de espera promedio.
- **Desventaja:** puede causar inanición de procesos con ráfagas largas.

## Escenario de ejemplo

Procesos: P1 (llegada 0, ráfaga 5), P2 (llegada 1, ráfaga 2), P3 (llegada 2, ráfaga 4), P4 (llegada 3, ráfaga 1).

Diagrama de Gantt esperado: P1[0–5], P4[5–6], P2[6–8], P3[8–12]
