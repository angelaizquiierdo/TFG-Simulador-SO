import { SimulationProvider } from 'cpu-scheduler';

## Descripción

El algoritmo de **Prioridad no expropiativa** selecciona el proceso con menor número
de prioridad (menor número = mayor prioridad). Una vez que un proceso entra en CPU,
la ocupa hasta terminar su ráfaga.

Si dos procesos tienen la misma prioridad, el desempate se resuelve por menor
`arrival_time` y luego por menor `id`.

## Escenario de ejemplo

<SimulationProvider
  algorithm="priority-np"
  processes={[
    { id: 'P1', arrival_time: 0, burst_time: 3, priority: 3 },
    { id: 'P2', arrival_time: 1, burst_time: 2, priority: 2 },
    { id: 'P3', arrival_time: 2, burst_time: 2, priority: 2 },
  ]}
  client:only="react"
/>
