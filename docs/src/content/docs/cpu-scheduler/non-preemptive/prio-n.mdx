import { SimulationProvider } from 'cpu-scheduler';

## Descripcion

El algoritmo de **Prioridad no expropiativo** selecciona el proceso con menor numero de
prioridad entre los que estan en la cola lista cuando la CPU queda libre. Un numero
menor indica mayor prioridad (p. ej., prioridad 1 es mas urgente que prioridad 3).

Es no expropiativo: una vez que un proceso entra en CPU, la retiene hasta finalizar
su rafaga, aunque llegue un proceso de mayor prioridad.

- Util cuando los procesos tienen niveles de urgencia definidos.
- Puede causar inanicion de procesos de baja prioridad.
