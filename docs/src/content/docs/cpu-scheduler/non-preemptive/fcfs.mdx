## Descripción

**FCFS** (First Come, First Served) es el algoritmo de planificación más sencillo. Los procesos se ejecutan en el orden en que llegan a la cola de listos. Es **no expropiativo**: una vez que un proceso ocupa la CPU, la mantiene hasta que termina.

- **Ventaja:** simple, sin inanición.
- **Desventaja:** puede producir el efecto convoy (un proceso largo bloquea a todos los que llegaron después).

## Escenario de ejemplo

Procesos: P1 (llegada 0, ráfaga 3), P2 (llegada 2, ráfaga 2), P3 (llegada 1, ráfaga 4).

Diagrama de Gantt esperado: P1[0–3], P3[3–7], P2[7–9]
