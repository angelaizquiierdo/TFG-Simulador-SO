## Props de `SimulationProvider`

| Prop | Tipo | Requerido | Descripción |
|------|------|-----------|-------------|
| `algorithm` | `string` | Sí | Nombre del algoritmo registrado (p. ej. `"fcfs"`, `"round-robin"`) |
| `processes` | `Process[]` | Sí | Lista de procesos a simular |
| `params` | `{ quantum?: number }` | No | Parámetros opcionales; `quantum` es requerido por Round Robin |
| `children` | `ReactNode` | No | Layout personalizado; si se omite, se usan los cuatro componentes por defecto |

## Tipo `Process`

```ts
interface Process {
  id: string;           // Identificador único
  arrival_time: number; // Tick de llegada (≥ 0)
  burst_time: number;   // Duración de la ráfaga (> 0)
  priority?: number;    // Prioridad (menor número = mayor prioridad); requerida por algoritmos de prioridad
}
```

## Ejemplos

### FCFS sin prioridad ni quantum

```mdx
<SimulationProvider
  algorithm="fcfs"
  processes={[
    { id: 'P1', arrival_time: 0, burst_time: 4 },
    { id: 'P2', arrival_time: 1, burst_time: 2 },
  ]}
  client:only="react"
/>
```

### Round Robin con quantum

```mdx
<SimulationProvider
  algorithm="round-robin"
  processes={[
    { id: 'P1', arrival_time: 0, burst_time: 5 },
    { id: 'P2', arrival_time: 1, burst_time: 4 },
  ]}
  params={{ quantum: 2 }}
  client:only="react"
/>
```

### Prioridad con campo `priority`

```mdx
<SimulationProvider
  algorithm="priority-np"
  processes={[
    { id: 'P1', arrival_time: 0, burst_time: 3, priority: 3 },
    { id: 'P2', arrival_time: 1, burst_time: 2, priority: 1 },
  ]}
  client:only="react"
/>
```

## Errores de configuración

- `burst_time = 0` → el contexto expone `error: "La ráfaga debe ser mayor que 0"`. No se ejecuta la simulación.
- `processes = []` → estado vacío, sin error.
- `algorithm` no registrado → el contexto expone un error descriptivo.
