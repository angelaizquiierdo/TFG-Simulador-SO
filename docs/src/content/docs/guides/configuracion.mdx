## Props de `SimulationProvider`

| Prop | Tipo | Requerida | Descripción |
|------|------|-----------|-------------|
| `algorithm` | `string` | Sí | Nombre del algoritmo registrado (p. ej. `"fcfs"`, `"round-robin"`). |
| `processes` | `Process[]` | Sí | Lista de procesos a simular. |
| `params` | `{ quantum?: number }` | No | Parámetros adicionales del algoritmo. |

## Estructura de un proceso

```ts
interface Process {
  id: string;           // Identificador único del proceso
  arrival_time: number; // Tiempo de llegada (≥ 0)
  burst_time: number;   // Duración de la ráfaga (> 0)
  priority?: number;    // Prioridad (menor número = mayor prioridad)
}
```

## Parámetro `quantum`

Solo lo usan los algoritmos de tipo Round Robin (`preemptionMode: 'on-quantum'`).
Si el algoritmo lo requiere (`requires.quantum === true`), el campo es obligatorio:

```tsx
<SimulationProvider
  algorithm="round-robin"
  processes={[...]}
  params={{ quantum: 2 }}
/>
```

## Campo `priority`

Los algoritmos de prioridad (`priority-np`, `priority-p`) leen el campo `priority`
del proceso. Un número menor indica mayor prioridad. Si el campo no se incluye, el
proceso se trata con la prioridad más baja posible.

## Errores de configuración

| Situación | Comportamiento |
|-----------|----------------|
| `burst_time = 0` | El contexto expone `error: "La ráfaga debe ser mayor que 0"` y no ejecuta la simulación. |
| `processes` vacío | Estado vacío sin error. |
| Nombre de algoritmo desconocido | El contexto expone un error descriptivo con los algoritmos disponibles. |
