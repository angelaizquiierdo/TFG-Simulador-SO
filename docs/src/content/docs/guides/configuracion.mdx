## Estructura de un proceso

Cada proceso en el array `processes` sigue la interfaz `Process`:

```ts
interface Process {
  id: string;           // Identificador unico del proceso
  arrival_time: number; // Tick en el que llega (>= 0)
  burst_time: number;   // Duracion de la rafaga (> 0)
  priority?: number;    // Prioridad (requerida por algoritmos de prioridad; menor = mas urgente)
}
```

## Algoritmos disponibles

| Nombre (`algorithm`) | Tipo | Requiere `priority` | Requiere `quantum` |
|---|---|---|---|
| `"fcfs"` | No expropiativo | No | No |
| `"sjf"` | No expropiativo | No | No |
| `"ljf"` | No expropiativo | No | No |
| `"priority-np"` | No expropiativo | Si | No |
| `"srtf"` | Expropiativo | No | No |
| `"priority-p"` | Expropiativo | Si | No |
| `"round-robin"` | Expropiativo | No | Si |

## Parametro quantum (Round Robin)

Round Robin requiere el parametro `quantum` en la prop `params`:

```mdx
<SimulationProvider
  algorithm="round-robin"
  processes={[...]}
  params={{ quantum: 2 }}
  client:only="react"
/>
```

El valor de `quantum` debe ser un entero positivo. Si se omite, el motor lanzara un error
en el momento de la simulacion.

## Metricas calculadas

El simulador calcula automaticamente las siguientes metricas para cada proceso:

| Metrica | Descripcion |
|---|---|
| `completion` | Tick en el que el proceso termina |
| `turnaround` | `completion - arrival_time` |
| `waiting` | `turnaround - burst_time` |
| `response` | Primer tick en CPU - `arrival_time` |

Y metricas agregadas:

| Metrica | Descripcion |
|---|---|
| `avgWaiting` | Promedio de `waiting` entre todos los procesos |
| `avgTurnaround` | Promedio de `turnaround` entre todos los procesos |
| `cpuUtilization` | Fraccion de ticks con CPU activa |
| `throughput` | Procesos completados por tick total |

## Casos especiales

- **Sin procesos:** el simulador devuelve un historial vacio sin error. El componente muestra un estado vacio.
- **`burst_time = 0`:** el motor lanza el error `"La rafaga debe ser mayor que 0"`. El componente muestra el mensaje de error.
