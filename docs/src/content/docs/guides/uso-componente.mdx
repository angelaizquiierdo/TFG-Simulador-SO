---
title: Cómo usar el componente
description: Guía para integrar el simulador de planificación de CPU en tu proyecto.
---

El simulador se distribuye como módulo TypeScript + React. Esta guía explica cómo incorporarlo en una página.

## Instalación

El paquete se importa como dependencia desde el directorio raíz del monorepo. En un proyecto externo, instálalo desde npm:

```bash
npm install cpu-scheduler
```

## Uso básico

El componente principal es `SimulationProvider`. Envuelve a los componentes visuales y les proporciona el contexto de simulación mediante React Context.

```tsx
import {
  SimulationProvider,
  ProcessTable,
  GanttChart,
  PlaybackControls,
  MetricsTable,
} from 'cpu-scheduler';

export default function Demo() {
  const processes = [
    { id: 'P1', arrival_time: 0, burst_time: 3 },
    { id: 'P2', arrival_time: 2, burst_time: 2 },
    { id: 'P3', arrival_time: 1, burst_time: 4 },
  ];

  return (
    <SimulationProvider algorithm="fcfs" processes={processes}>
      <ProcessTable />
      <GanttChart />
      <PlaybackControls />
      <MetricsTable />
    </SimulationProvider>
  );
}
```

## Componentes disponibles

| Componente | Descripción |
|---|---|
| `SimulationProvider` | Ejecuta la simulación y expone el contexto |
| `ProcessTable` | Tabla de procesos de entrada |
| `GanttChart` | Diagrama de Gantt en forma de matriz |
| `PlaybackControls` | Controles de reproducción (play, pause, paso) |
| `MetricsTable` | Métricas por proceso y agregadas (visibles al final) |

## Hook `useSimulation`

Si necesitas acceder al contexto directamente desde un componente propio:

```tsx
import { useSimulation } from 'cpu-scheduler';

function MiComponente() {
  const { tick, result, atEnd } = useSimulation();
  return <p>Tick actual: {tick}</p>;
}
```

`useSimulation` lanza un error si se usa fuera de un `SimulationProvider`.
