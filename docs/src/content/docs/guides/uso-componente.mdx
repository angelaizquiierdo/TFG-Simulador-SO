# Cómo usar el componente `<Simulator>`

El componente `<Simulator>` muestra una simulación interactiva de planificación de CPU
con diagrama de Gantt, controles de reproducción y tabla de métricas.

## Instalación

```bash
npm install cpu-scheduler
```

## Uso básico

```tsx
import { Simulator } from 'cpu-scheduler';

export default function MiPagina() {
  return (
    <Simulator
      algorithm="fcfs"
      processes={[
        { id: 'P1', arrival_time: 0, burst_time: 3 },
        { id: 'P2', arrival_time: 2, burst_time: 2 },
      ]}
    />
  );
}
```

## Props obligatorias

| Prop | Tipo | Descripción |
|------|------|-------------|
| `algorithm` | `string` | Nombre del algoritmo registrado (p. ej. `"fcfs"`, `"round-robin"`) |
| `processes` | `Process[]` | Lista de procesos a simular |

## Comportamiento con lista vacía

Si `processes` es un array vacío, el componente muestra un estado vacío sin lanzar ningún error.

## Comportamiento con configuración inválida

Si algún proceso tiene `burst_time ≤ 0`, se muestra el mensaje
`"La ráfaga debe ser mayor que 0"` y no se ejecuta la simulación.
